# ⬡ RTSP Viewer

A browser-based RTSP camera viewer. Connect to any IP camera and watch the live feed directly in your browser — no plugins, no apps needed.

Built with **mediamtx** (RTSP → WebRTC converter) + vanilla HTML/CSS/JS.

---

## 🚀 Quick Start — Transfer to Another Machine

**Copy these 6 files to your new machine:**
```
index.html
style.css
app.js
mediamtx.yml
setup.sh
start.sh
```

**Run these commands on the new machine:**
```bash
# Install dependencies (one-time)
sudo apt update
sudo apt install -y python3 wget tar ffmpeg

# Setup and start
chmod +x setup.sh start.sh
./setup.sh
./start.sh

# Then open in browser
http://localhost:3000
```

**From another device on the network:**
```
http://NEW_MACHINE_IP:3000
```
Set **MediaMTX Host** field to `NEW_MACHINE_IP` in the viewer.

Find your machine IP with:
```bash
hostname -I | awk '{print $1}'
```

### ✅ Transfer Readiness Checklist

Before moving to another machine, confirm:

- `setup.sh` and `start.sh` are executable
- `mediamtx.yml` has your final camera settings
- You are transferring the 6 required files listed above
- You will run `./setup.sh` on the target machine (required)

Important:

- Do **not** rely on copying the `mediamtx` binary between different CPU architectures
- Always run `./setup.sh` on the new machine so it downloads the correct binary

Optional: create a single archive for transfer:

```bash
tar -czf rtsp-viewer-transfer.tar.gz index.html style.css app.js mediamtx.yml setup.sh start.sh README.md
```

## How It Works

```
IP Camera (RTSP)  ──►  mediamtx  ──►  WebRTC  ──►  Browser
                       (running on your PC/Pi)
```

Browsers cannot play RTSP streams directly. mediamtx runs on your machine, pulls the RTSP stream from the camera, and converts it to WebRTC which any modern browser can play — with only ~100–300ms latency.

### Step-by-Step Installation

#### 1. Install System Dependencies

```bash
sudo apt update
sudo apt install -y python3 wget tar ffmpeg
```

This installs:
- **python3** — runs the web server on port 3000
- **wget** — downloads MediaMTX in `setup.sh`
- **tar** — extracts MediaMTX archive
- **ffmpeg** — only needed for testing with fake streams

#### 2. Download Project Files

**Option A: Git clone**
```bash
git clone https://github.com/arman4h/rtsp-webviewer.git
cd rtsp-viewer
```

**Option B: Manual download**
```bash
# Download the zip file, then:
unzip rtsp-viewer.zip
cd rtsp-viewer
```

**Option C: Copy from another machine**
Copy these 6 files to your new machine:
- `index.html`, `style.css`, `app.js`
- `mediamtx.yml`, `setup.sh`, `start.sh`

#### 3. Make Scripts Executable

```bash
chmod +x setup.sh start.sh
```

#### 4. Run Setup (One-Time)

```bash
./setup.sh
```

This:
- Detects your CPU architecture (x86_64, aarch64, armv7l, armv6l)
- Downloads the correct MediaMTX binary from GitHub
- Preserves your existing `mediamtx.yml` configuration
- Creates the `mediamtx` executable binary

#### 5. Start the Viewer

```bash
./start.sh
```

You should see:
```
Starting RTSP Viewer...
mediamtx started (PID 12345)
INF [RTSP] listener opened on :8554
INF [WebRTC] listener opened on :8889
INF [API] listener opened on :9997
Web server started (PID 12346)

Open: http://localhost:3000
Press Ctrl+C to stop
```

#### 6. Open in Browser

**Same machine (localhost):**
```
http://localhost:3000
```

**From another device on the same LAN:**
```
http://YOUR_MACHINE_IP:3000
```

To find your machine IP:
```bash
hostname -I | awk '{print $1}'
```

## Connecting a Real IP Camera

### OpenIPC Camera

Default RTSP URL format:
```
rtsp://username:password@CAMERA_IP:554/stream=0
```

Fill the form:

| Field | Value |
|---|---|
| Camera IP | Your camera's IP (e.g. `192.168.1.10`) |
| Port | `554` |
| Username | Your camera username |
| Password | Your camera password |
| Stream Path | `stream=0` (main) or `stream=1` (sub) |
| mediamtx Host | IP of the machine running mediamtx |

### Generic IP Camera (CCTV)

Common stream paths to try:

| Brand | Stream Path |
|---|---|
| Hikvision | `Streaming/Channels/101` |
| Dahua | `cam/realmonitor?channel=1&subtype=0` |
| Generic | `stream=0` or `ch0` or `live` |

#### Find Stream Path from Camera Manual

1. Open your camera's web interface: `http://CAMERA_IP`
2. Look for settings called "Stream", "RTSP", "Live", or "Encoding"
3. Find the stream path (often `/stream=0`, `/live0`, or `/ch1`)
4. Fill in the form with these values

## Important Setup Notes

- Run `./setup.sh` only when you first install or update MediaMTX.
- `setup.sh` now preserves your existing `mediamtx.yml` so camera settings are not lost.
- Do not start a second `mediamtx` process manually while `./start.sh` is already running.
- If you want another laptop/phone to view the stream, use the LAN IP of the machine running this project, not `localhost`.

### Rover / Vehicle Deployment

For rover use, run this project on the rover computer itself:

- Camera → rover computer → MediaMTX → browser on controller laptop
- Keep the rover computer and controller laptop on the same LAN or a dedicated 5 GHz Wi‑Fi link
- Prefer Ethernet for camera-to-rover if possible
- Use the rover machine IP in the browser URL: `http://ROVER_IP:3000`
- Set **MediaMTX Host** to `ROVER_IP`

### Low-Latency Camera Tuning

The biggest latency improvements come from the camera settings, not the browser:

- H.264 only
- B-frames off
- Keyframe interval: 1 second or lower
- Resolution: 640x480 or 1280x720
- Bitrate: about 1–3 Mbps
- Enable the camera's low-latency / real-time mode if available

### Expected Latency

With a browser viewer, **80–150 ms** is normal on a good LAN. Lower than that is difficult in WebRTC browser playback, even when the network is local.

### If You Move the Project to Another Ubuntu Machine

**See "Quick Start — Transfer to Another Machine" at the top.**

In short:
1. Copy the 6 files listed in Quick Start to the new machine
2. Run `sudo apt install -y python3 wget tar ffmpeg`
3. Run `chmod +x setup.sh start.sh`
4. Run `./setup.sh` (downloads MediaMTX for this machine's CPU architecture)
5. Run `./start.sh`
6. Open `http://localhost:3000` or `http://MACHINE_IP:3000` from another device

Your `mediamtx.yml` camera settings are preserved and will work immediately on the new machine.

## Troubleshooting

### White screen / page not loading
```bash
# Make sure web server is running
python3 -m http.server 3000
# Then open http://localhost:3000
```

### "Stream not found" error in browser
- Make sure mediamtx is running (`./mediamtx`)
- Make sure the Stream Path in the form matches what mediamtx has
- Check mediamtx terminal for errors
- Verify stream is active:
```bash
curl http://localhost:9997/v3/paths/list
```

### Black video but "Connected" status
- Click directly on the black video area to trigger play
- Browser autoplay policy blocks video until user interacts

### Cannot reach camera
```bash
# Test if camera is reachable on network
ping CAMERA_IP

# Test RTSP stream directly with ffplay
ffplay rtsp://username:password@CAMERA_IP:554/stream=0
```

### Open viewer from another device

Use the rover or host machine IP, not localhost:

```text
http://ROVER_IP:3000
```

Then set **MediaMTX Host** to the same `ROVER_IP`.

### Camera and PC on different IP subnets

This is a common issue when connecting directly to an IP camera via Ethernet cable. The camera may have a fixed IP like `192.168.1.10` but your PC's Ethernet port may be on a different subnet like `192.168.0.x` — they won't be able to talk to each other.

**Check your PC's current Ethernet IP:**
```bash
ip addr show
```

Look for the `inet` line under your Ethernet interface (usually `eth0` or `enp3s0`). Example output:
```
inet 192.168.0.55/24   ← your PC is on 192.168.0.x
```

If the camera is on `192.168.1.10` but your PC is on `192.168.0.x`, they are on different subnets and cannot communicate.

**Fix — Change your PC's Ethernet IP to match the camera's subnet:**

```bash
# Find your Ethernet interface name first
ip link show

# Set your PC's IP to the same subnet as the camera
# Example: camera is 192.168.1.10, so set PC to 192.168.1.X (any number except 10)
sudo ip addr add 192.168.1.45/24 dev eth0

# Replace eth0 with your actual interface name (e.g. enp3s0, ens33)
```

Now test:
```bash
ping 192.168.1.10
```

**Note:** This IP change is temporary and resets after reboot. To make it permanent, use your network manager:

```bash
# Using nmcli (NetworkManager)
nmcli con mod "Wired connection 1" ipv4.addresses 192.168.1.50/24
nmcli con mod "Wired connection 1" ipv4.method manual
nmcli con up "Wired connection 1"
```

Or via GUI: **Settings → Network → Wired → Gear icon → IPv4 → Manual → set Address to `192.168.1.50`, Netmask to `255.255.255.0`**

**General rule:** Your PC's IP and the camera's IP must share the same first three numbers:

| Camera IP | Your PC must be | ✓/✗ |
|---|---|---|
| `192.168.1.10` | `192.168.1.X` | ✓ Same subnet |
| `192.168.1.10` | `192.168.0.X` | ✗ Different subnet |
| `10.0.0.5` | `10.0.0.X` | ✓ Same subnet |

### mediamtx won't start
```bash
# Check if ports are already in use
ss -tlnp | grep -E '8554|8889|9997'

# Kill any existing mediamtx instance
pkill mediamtx
```

### Permission denied on scripts
```bash
chmod +x start.sh setup.sh mediamtx
```

## Stop the Server

Press `Ctrl+C` in the terminal running `start.sh`. It stops both mediamtx and the web server automatically.

Or stop manually:
```bash
pkill mediamtx
pkill -f "http.server 3000"
```

## 📖 Complete Command Reference

### First-Time Setup

```bash
# 1. Install system packages
sudo apt update
sudo apt install -y python3 wget tar ffmpeg

# 2. Download project (if not already downloaded)
git clone https://github.com/arman4h/rtsp-webviewer.git
cd rtsp-viewer

# 3. Make scripts executable
chmod +x setup.sh start.sh

# 4. Download MediaMTX binary (detects your CPU automatically)
./setup.sh
```

### Running the Viewer

```bash
# Start everything
./start.sh

# Stop everything (also try Ctrl+C)
pkill mediamtx
pkill -f "http.server"

# Kill stale processes if viewer won't start
pkill -x mediamtx 2>/dev/null || true
./start.sh
```
